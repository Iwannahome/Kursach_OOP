ГОСТ Единой системы программной документации (ЕСПД) [1]. Все этапы

программирование» [2-3] и методике разработки объектно-ориентированных
программ [4-6].
Объектно-ориентированное программирование- это парадигма разработки
программирования, в которой основной концепцией является понятие объекта,
который отождествляется с предметной областью. Объект - отдельный
представитель класса, имеющий конкретное поведение и состояние, полностью
определяемое классом. Объект имеет конкретные значения атрибутов и методы,
которые работают с этими значениями на основе заданнных в классе правил.[1]
Появление ООП обусловливалось необходимостью решения возникающих
проблем при построении сложных систем. ООП позволяет использовать один и
тот же программный код с разными данными. На основе классов создаётся
множество объектов, у каждого из которых могут быть собственные значения
полей. Нет необходимости вводить множество переменных. Наследование
позволяет не писать новый код. Основные принципы ООП: Абстракция - набор
значимых характеристик объекта, исключая из рассмотрения незначимых.
Полиморфизм - свойство системы использовать объекты с одинаковым
интерфейсом без информации о типе внутренней структуре объекта.
Наследование - свойство системы, которое позволяет описать новый класс на
основе уже существующего с частично или полностью заимствующейся
функциональностью. Инкапсуляция - свойство системы, позволяющее объединить
